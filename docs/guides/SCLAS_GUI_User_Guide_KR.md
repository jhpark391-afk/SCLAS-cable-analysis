# [User Guide] HELIX / SCLAS GUI 통합 사용 설명서

본 설명서는 해저 케이블 구조 해석 자동화 플랫폼 **HELIX / SCLAS** GUI의 모든 화면 구성, 입력 필드, 제어 버튼 및 그래프 도구의 사용법을 초보자 눈높이에 맞춰 설명한 종합 사용자 매뉴얼입니다.

---

## 🧭 1. 프로그램 기동 및 언어 설정

* **기동 방법 (Windows)**: 
  * 바탕화면이나 루트의 `SCLAS_Quick_Launch/` 폴더에 있는 **`run_sclas.bat`** 파일을 더블클릭하면 명령창(CMD)이 켜진 후 GUI 화면이 즉시 나타납니다.
* **언어 전환 버튼 (English / 한국어 Toggle)**:
  * GUI 상단 우측 배너 영역에 있는 **`[English / 한국어]`** 버튼을 클릭하면, 화면의 모든 라벨, 경고 메시지, 진단 보고서 텍스트가 실시간으로 국/영문 전환됩니다.

---

## 📐 2. Design (설계 변수 입력) 탭

케이블을 구성하는 각Annular(고리형) 레이어의 치수와 아머 와이어(Armour Wire) 배치 각도를 입력하는 전처리 단계입니다.

### 2.1. 입력 항목 가이드 (Input Parameters)
1. **도체 코어 반경 (Core Radius, mm)**: 케이블 정중앙에 위치하는 전력선/광섬유 핵심 도체 다발의 외반경입니다.
2. **시스(Sheath) 두께 (Sheath Thickness, mm)**: 코어 및 아머층을 감싸는 폴리머 보호 피복의 두께입니다.
3. **Bedding 두께 (Bedding Thickness, mm)**: 내/외측 아머 와이어 레이어 사이에 위치하여 쿠션 역할을 하는 완충층의 두께입니다.
4. **아머 와이어 반경 (Armour Wire Radius, mm)**: 케이블 인장력 지탱을 위해 나선형으로 감기는 금속 보강재(아머)의 단면 반경입니다.
5. **아머 와이어 조립 각도 (Lay Angle, deg)**: 케이블 길이 방향 축 기준 아머 와이어가 꼬여 들어가는 나선 각도입니다. 이 각도가 클수록 와이어가 조밀하게 감기며 굽힘 강성이 달라집니다.
6. **아머 와이어 개수 (Wire Count)**: 단면에 원주 방향으로 조립되는 금속 와이어의 총 개수입니다.

### 2.2. 실시간 계산 배너 (Derived Geometry Summary)
* 치수 입력창 바로 옆에 위치한 노란색/연청색 정보 배너입니다.
* 입력된 수치를 기반으로 아바쿠스가 인식해야 할 내/외측 아머 레이어의 실제 조립 반경과 단면 강성 값(EI)을 **아바쿠스 구동 전에 실시간으로 자동 계산하여 표출**해 줍니다. 수치가 너무 비현실적이면 빨간 경고창이 뜹니다.

## 🕸️ 3. Mesh (유한요소 격자 생성) 탭

3차원 기하학 구조를 유한요소 해석용 격자(Mesh) 모델로 분할하기 위한 수치 해석 설정 단계입니다.

### 3.1. 격자 조밀도 설정 (Mesh Density Settings)
1. **축 방향 분할수 (Axial Divisions)**: 케이블 전체 길이 방향으로 메쉬를 얼마나 조밀하게 쪼갤 것인지 정의합니다. (기본값: 가벼운 테스트 시 4~10분할, 정밀 해석 시 40분할 이상)
2. **코어 원주 방향 분할수 (Core Circumferential Divisions)**: 단면 내 폴리머 레이어를 원주 방향으로 몇 조각 낼 것인지 정의합니다.
3. **아머 단면 분할수 (Armour Wire Segment Divisions)**: 원형 아머 와이어 단면의 둥근 테두리를 몇 개의 세그먼트로 나눌 것인지 지정합니다.

### 3.2. 모델링 전략 선택 (Modeling Strategy Combo-Box)
* **Beam with Contact Surface (추천)**: 
  * 아머 와이어를 1차원 보(Beam, B31) 요소로 모델링하되 표면에 가상의 접촉 면을 입혀 연산 속도와 접촉 물리 정밀도를 동시에 챙기는 표준 기법입니다.
* **Solid Element (정밀)**: 
  * 모든 와이어를 3D 입체 요소(Solid)로 온전히 모델링합니다. 연산 시간이 크게 늘어나지만 가장 엄밀한 파손 해석이 가능합니다.

### 3.3. 주요 버튼 기능
* **`[Generate Mesh Preview]` (격자 미리보기)**:
  * 입력된 분할수에 맞춰 아바쿠스 인풋이 문제없이 생성 가능한 기하학적 형상인지 가상 단면 메쉬를 임시 연산하여 프리뷰 리스트에 채워줍니다.
* **`[Export Mesh Deck]` (격자 데이터 내보내기)**:
  * 현재의 메쉬 설정을 `input_data.json` 및 백엔드 지시서인 `abaqus_mesh_manifest.json` 형태로 물리적으로 생성 및 저장합니다.

## 📊 4. Analysis (해석 실행 및 가시화) 탭

아바쿠스 솔버를 실제로 가동하고, 획득한 이력 곡선을 관찰 및 다각도로 비교 분석하는 후처리 핵심 화면입니다.

### 4.1. 해석 조건 입력 (Solver Conditions)
1. **최대 곡률 (Max Curvature, 1/m)**: 케이블을 강제로 얼마나 구부릴 것인지 굽힘 가혹도를 정의합니다.
2. **마찰 계수 (Friction Coefficient)**: 금속 와이어와 폴리머 레이어 간의 마찰 상수로, 모멘트 이력 루프의 폭(소실 에너지량)을 결정합니다.
3. **접촉 Penalty 강성 (Beta)**: 비선형 접촉 해석의 수렴 안정성을 지휘하는 인자입니다. 해석이 자꾸 수렴 실패하면 이 값을 미세 조정합니다.
4. **반복 사이클 수 (Loading Cycles)**: 구부렸다 폈다를 몇 번 반복할 것인지 설정합니다. (보통 1.5~2사이클에서 완전한 마찰 이력 루프가 관찰됨)

### 4.2. 중앙 그래프 창 (PyQtGraph Plotter)
* **기능**: 해석이 끝난 모멘트-곡률 선도(Hysteresis Loop)를 실시간으로 렌더링합니다.
* **인터랙션 조작법**:
  * **줌 인/아웃**: 마우스 휠을 굴리거나 마우스 우클릭 상태로 드래그합니다.
  * **전체 화면 복원**: 그래프 안에서 마우스 더블클릭 혹은 우클릭 메뉴의 `Auto Scale`을 선택합니다.
  * **영역 확대**: 마우스 좌클릭 드래그로 박스를 그리면 해당 영역이 확대됩니다.

### 4.3. 하단 제어 버튼 및 배너 기능 상세

| 버튼명 / 배너명 | 상세 역할 및 초보자 팁 |
| :--- | :--- |
| **`[Run / Create Job]`** | **핵심 실행 버튼**. 누르면 아바쿠스가 noGUI 모드로 기동되어 솔버 연산을 시작합니다. 완료 즉시 그래프가 자동으로 갱신됩니다. |
| **`[Load CSV for Comparison]`** | 과거 해석안이나 문헌 시험치(CSV 파일)를 불러와, 현재 구동된 그래프 위에 **점선이나 얇은 선 레이어로 중첩(Overlay)**하여 설계 오차를 시각적으로 즉각 비교하게 해 줍니다. |
| **`[Clear Comparison]`** | 중첩해서 띄운 비교용 외부 그래프 레이어들을 차트에서 깨끗이 지우고 현재 데이터만 남깁니다. |
| **`[Export Chart as PNG]`** | 현재 그려진 모멘트-곡률 그래프 화면을 보고서 첨부용 고화질 PNG 이미지 파일로 즉시 캡처하여 저장합니다. |
| **`[Save Current Data as CSV]`** | 현재 해석되어 화면에 뿌려진 곡률 및 굽힘 모멘트 수치 테이블 전체를 외부 가공이 가능하도록 `result_data.csv` 형태로 내보냅니다. |
| **`[Focus Chart]`** | 입력창 사이드바를 화면에서 숨기고 그래프를 모니터 전체 화면으로 넓게 확대하여 세밀한 차트 분석을 돕습니다. (다시 누르면 원복) |

### 4.4. 우측 정보 및 오프라인 진단 패널 (Recent Jobs & Diagnostics)
1. **최근 작업 목록 (Recent Jobs)**:
   * 저장소의 `jobs/SCLAS_jobs/` 폴더 내에 누적된 과거 해석 기록들의 날짜와 상태(PASS/FAIL)를 실시간 리스트업해 줍니다. 
   * 리스트에서 임의의 과거 항목을 클릭하면 아바쿠스를 새로 돌리지 않고도 **과거의 모멘트 그래프가 즉시 화면에 복원**됩니다.
2. **`[Diagnose selected]` (선택된 작업 진단)**:
   * 해석 수행 중 '비수렴'이나 '크래시'가 났을 때, 해당 실패 폴더를 클릭하고 이 버튼을 누르면 아바쿠스 엔진 내부 로그 파일의 실패 사유를 역추적하여 **"Penalty 수정 필요", "Mesh 수정 필요"** 등의 원포인트 처방 가이드를 빨간색 진단 배너에 상세히 출력해 줍니다.

## 💡 5. 초보 설계자를 위한 꿀팁 (Quick Reference)

* **해석 속도를 빠르게 하고 싶을 때**:
  * `Mesh` 탭에서 **`Axial Divisions`를 4~8 내외**로 아주 낮추고, **`Loading Cycles`를 1**로 맞춘 뒤 해석을 가동하십시오. 단 1분 만에 대략적인 모멘트 거동 선도가 완성되어 레이아웃 검토가 빨라집니다.
* **그래프 형상이 꺾이지 않고 일직선으로만 나올 때**:
  * 마찰이 전혀 작용하지 않는 상태이거나 메쉬 크기가 너무 커서 슬립이 일어나지 않는 상태입니다. `Analysis` 탭의 `Friction Coefficient`를 `0.1~0.3` 사이로 올리고 격자를 좀 더 촘촘히 나누어 보십시오.
* **해석이 중간에 에러(FAIL)를 내며 멈출 때**:
  * 당황하지 말고 우측 최근 작업에서 해당 FAIL 폴더를 클릭 후 **`[Diagnose selected]`**를 누르십시오. 로그 내용 하단에 표시되는 붉은 가이드라인의 수치 제안대로 설정을 수정한 뒤 다시 돌리면 수렴에 성공합니다.
